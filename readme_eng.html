Method A

가장 기본적인 속성 설정 방법입니다.
아래의 속성표를 참고하여 다양한 기능을 설정할 수 있습니다.


속성 및 기능 표
data-check Set the type of validation.
The value of the name attribute of the element to set the Validation
(if set to data-validation attribute in HTML, it is not required)
char, eng, kor, lang, email, num, money, file, form
data-minlength Set the minimum letters
data-maxlength Set the maximum letters
data-mincheck Set the minimum number of checks on a checkbox
data-maxcheck Set the maximum number of checks on a checkbox
data-minvalue minimum value
data-maxvalue maximum value
data-maxsize set the maximum file size for upload
(after the number, you are able to use units just like ‘kb,mb,gb,tb’)
data-filetype set the type of file extension for upload
data-group When users fill in a group form like phone number, they have to pass the validation check.
data-required Set the necessary information.
data-format format for a user to set

※ Data Type
n:number, c:character, a:all character,
yyyy:years, mm:month, dd:day from 01 to 31,
hh: hours from 00 to 23, mi:minutes from 00 to 59, ss:second from 00 to 59

※ Divider
데이터 타입으로 규정된 알파벳(ncaymdhis)을 제외한
나머지 모든 문자는 구분자로서 자동입력됩니다.
data-no-auto-format Canceling a divider which automatically fills in the added format.

Form
check속성을 form으로 입력하면, submit버튼으로 데이터를 전송할 때, 자동으로 sjvalidator가 설정된 모든 FORM태그의 유효성검사를 실시 합니다.

Submit


JAVASCRIPT로 submit()을 호출할 경우, onsubmit()이벤트를 거치지 않습니다.
떄문에 자동 유효성검사를 실시하지 않게 됩니다. 그때는 다음과 같이 해주시면 해결가능합니다.
if(empForm.onsubmit()){empForm.submit()};
※ empForm은 FORM태그의 name속성 값
