2. 속성 및 기능

sjvalidator를 적용하는 방법은 세 가지 방법이 있습니다.
각각 장점과 단점이 존재하므로 상황에 가장 적합한 방식을 하나 선택하면 됩니다.


에러메세지가 바로 아래에 출력되기 위해서는 div또는 span태그 등으로 감싸줘야 합니다.
(어렵게 말해서 ErrorMessage는 sjvalidator가 설정된 Form태그(input, select, textarea)의 부모노드의 막내로서 생기게 됩니다.)

방법 A

가장 기본적인 속성 설정 방법입니다.
아래의 속성표를 참고하여 다양한 기능을 설정할 수 있습니다.


속성 및 기능 표
data-check 유효성 검사형식을 지정합니다.
char, eng, kor, lang, email, num, money, file, form
data-minlength 최소 글자수 설정
data-maxlength 최대 글자수 설정
data-mincheck checkbox 최소 체크수
data-maxcheck checkbox 최대 체크수
data-minvalue 최소값
data-maxvalue 최대값
data-maxsize 업로드 가능한 파일의 최대용량 제한
(숫자 뒤에 kb, mb, gb, tb 단위 사용가능)
data-filetype 업로드 가능한 파일의 확장자
data-group 원하는 그룹명을 적으면, 같은 그룹으로 묶인 요소들은 모두 유효성에 통과해야 함.
data-required 필수입력사항 설정
data-format 사용자 지정 형식

※ 데이터 타입
n:숫자, c:문자, a:모든문자,
yyyy:년도, mm:01~12의 숫자, dd:01~31의 숫자,
hh:00~23의 숫자, mi:00~59의 숫자, ss:00~59의 숫자

※ 구분자
데이터 타입으로 규정된 알파벳(ncaymdhis)을 제외한
나머지 모든 문자는 구분자로서 자동입력됩니다.
data-no-auto-format 사용자 지정 형식의 구분자를 자동입력 해제.
data-validator 위의 속성들을 일괄처리할 수 있습니다. ※방법 B를 참고

Submit


JAVASCRIPT로 submit()을 호출할 경우, onsubmit()이벤트를 거치지 않습니다.
떄문에 자동 유효성검사를 실시하지 않게 됩니다. 그때는 다음과 같이 해주시면 해결가능합니다.
if(empForm.onsubmit()){empForm.submit()};
※ empForm은 FORM태그의 name속성 값
